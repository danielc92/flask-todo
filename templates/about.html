Purpose of this application

This application was built to experiment with the interaction between a NOSQL database and a dynamic web application. Previously I've built web applications using a relational database backend and wanted to get a feel architecting and building around a NOSQL system.

I also wanted to build this tool with the aim of boosting productivity for those who wish to use it. This can be done through simplficiation of an agile-esque workflow, simple task/card system and easy to use GUI.

NOSQL has some notable advantages over SQL, in that it:

  • Scales well horizontally.
  • Relies on a document style system, similar to hashtable structure. This means that little to no transforming needs to be done in between reading and rendering data.
  • Need for complex joins are eliminated.
  • Faster development time.
